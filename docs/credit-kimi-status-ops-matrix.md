## 1 字段定义

| 字段 | 说明 | 取值 |
|------|------|------|
| **信用评分** | 信用风险量化评分，**分数越高风险越高** | 数值范围（见 §2 档位） |
| **信用等级** | 基于评分划分的等级标签 | 优秀 / 良好 / 一般 / 差 |
| **自动审核** | 系统自动审批结果 | 处理中 / 通过 / 拒绝 / 预警 |
| **人工审核** | 人工处置状态 | — / 待审核 / 提交复核 / 复核放行 / 复核拒绝 |
| **审核人** | 当前审核环节的审核员信息 | — / 初审：审核员 N / 初审：审核员 N；终审：主管 N |
| **操作** | 列表行可用的操作按钮（按置灰优先级排列） | 见下方矩阵 |

## 2 评分档位与等级

| 信用评分区间 | 信用等级 | 含义 |
|-------------|---------|------|
| 801 – 900 | 优秀 | 低风险，自动通过 |
| 651 – 800 | 良好 | 低风险，自动通过 |
| 501 – 650 | 一般 | 中风险，需人工介入 |
| 300 – 500 | 差 | 高风险，自动拒绝 |
| —（处理中） | — | 系统计算中，暂无评分 |

> 注：评分口径为「分数越高 = 风险越高」。子维度得分（身份真实性、还款能力等）口径相反（越高越好），汇总时经加权+惩罚转换为风险分。

## 4 操作按钮说明

| 按钮 | 触发条件 | 行为 |
|------|---------|------|
| **查看** | 所有非「处理中」状态均可点击 | 进入报告详情页 |
| **提交复核** | 预警 + 待审核 | 提交至双人复核流程 |
| **确认放行** | 预警 + 已提交复核（初审岗） | 审核通过，授信 |
| **确认拒绝** | 预警 + 已提交复核（初审岗） | 拒绝授信 |
| **录入备注** | 预警 + 待审核 或 已提交复核 | 添加审核备注 |

> 「查看」在「处理中」状态下**按钮置灰不可点击**（系统仍在计算，报告尚未生成）。

## 5 自动审核 → 人工审核 状态流转规则

```
处理中 ──计算完成──→ 通过 / 拒绝 / 预警
                          │        │       │
                        结束     结束   待审核
                                         │
                                    ├─── 提交复核 ──→ 复核放行 / 复核拒绝（结束）
                                    │
                                   录入备注（不改变状态）
```

- **通过 / 拒绝**：终态，无需人工干预，仅可查看。
- **预警**：进入人工审核队列，初始状态为「待审核」。
- **待审核** → 可「提交复核」或「录入备注」。
- **提交复核** → 进入双人复核，初审岗可「确认放行」或「确认拒绝」。
- **复核放行 / 复核拒绝**：终态。

## 6 与欺诈识别矩阵的差异

| 维度 | 信用风控（本表） | 欺诈识别（fraud-scheme4） |
|------|-----------------|------------------------|
| 评分名称 | 信用评分 | 欺诈风险分值 |
| 等级名称 | 优秀/良好/一般/差 | 低/中/高/极高 |
| 评分区间 | 300–900（4 档） | 0–100（4 档） |
| 高风险操作 | 确认放行/确认拒绝 | 加入黑名单/确认拒绝 |
| 低风险操作 | 仅查看 | 确认放行 |
| 处理中行为 | 查看置灰 | 查看置灰（一致） |
