## 3. 字段与状态语义说明

- **审核人列**：未确认 / 待审核 / 待确认 阶段为「—」（尚未分派）；确认或办结类显示「初审：审核员 1」；强制放行办结 / 加入黑名单 / 双人复核办结显示「初审：审核员 1；终审：主管 1」。
- **查看（按钮置灰）**：仅出现在「处理中 / 核验计算中」这一行，查看按钮禁用、不可点击。
- **报告确认**：复核员对系统初判结论做确认，不改变风险等级，写入操作日志。
- **强制复审 / 强制放行办结**：高风险初审拒绝后，经特殊通道强制放行，须终审主管审批；办结后仅可查看。
- **加入黑名单**：高 / 极高风险「待确认」可达；确认后纳入黑名单，须初审 + 终审主管两级审批；办结后仅可查看。
- **提交双人复核 / 确认放行 / 确认拒绝**：中风险「预警」走双人复核流程——初审提交 → 双审放行或拒绝办结（两级审批）。
- **录入备注**：中风险各阶段均可补充备注，不影响状态流转。
- **仅【查看】**：流程已办结（已确认 / 初审确认拒贷办结 / 强制放行办结 / 加入黑名单 / 双人复核办结），列表行只读。

## 4. 与详情页 RuleTable「权重—豁免」映射的关系

列表「操作」由「欺诈风险等级 + 人工审核」二维推导（见第 2 节矩阵）；详情页规则表「状态 / 豁免」则由单条规则**权重**推导，二者正交、互不影响：

| 规则权重 | 命中状态 | 豁免 |
|---|---|---|
| 极高 | 命中 | 不可豁免（仅「详情」） |
| 高 | 命中 | 豁免需审批 |
| 中 | 命中 | 可自免 |
| 低 | 命中 | 可自免 |
| 任意 | 未命中 | 未命中（无按钮） |

> 四档权重定义：0–39 低（绿）/ 40–59 中（黄）/ 60–79 高（橙）/ 80–100 极高（红）。
