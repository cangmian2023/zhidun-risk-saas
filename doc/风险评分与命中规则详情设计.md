# 风险评分与命中规则详情设计

> 整理日期：2026-07-19
> 位置：申贷审核详情页内的「风险评分」卡片与「命中规则与模型」卡片
> 目的：明确两部分的展开态内容，让复核员能看懂分数来源和规则命中逻辑，支撑人工复核决策。
> 
> **关联文档**：前端报告展示规范与 3 套完整样例数据见《申贷审核报告-风险评分与命中规则展示规范与样例数据.md》。

---

## 1. 设计原则

1. **结论先看到，过程可下钻**：卡片默认显示分数/命中结果，展开后展示推导过程。
2. **可解释，不泄露模型**：展示影响因子的方向和相对权重，不暴露模型系数、阈值、数据源细节。
3. **复核导向**：每条信息都要回答复核员的三个问题："这个分数为什么是这个数？""这条规则具体查了什么？""我需要据此做什么？"
4. **方向一致性（优化）**：三个分数方向并不统一（欺诈分越高越危险，信用分/综合分越高越安全）。详情页必须提供"统一方向"的头部风险指标，且每个原生分数明确标注方向，避免复核员误读。

---

## 2. 风险评分 · 展开态详情

### 2.1 通用结构（三个分数共用）

> **方向一致性（优化）**：风险评分卡片整体布局建议为"1 个统一头部指标 + 3 个原生分数拆解"。

**头部统一风险指标（越高越危险，单一方向）**
在卡片最上方放一个 **风险指数 / 风险等级（0~100，越高越危险）**，由三分数融合推导，方向统一，作为一眼结论，避免同时读三个反向指标导致的误判。

**原生分数卡片（每个标注方向）**
每个分数卡片展开后分为四区，头部须含**方向标识**：

```
┌─ 分数头部 ─────────────────────────────┐
│ 分数值 | 等级 | 区间 | 阈值线 | 方向标识 | 模型版本/计算时间 │
│ 例：欺诈分 标签"↑越高越危险"(红系)         │
│     信用分 标签"↑越高越优质"(绿系)         │
├─ 评分摘要 ─────────────────────────────┤
│ 一句话解释当前分数含义（业务口径）         │
├─ 影响因素/维度拆解 ─────────────────────┤
│ 各因子名称 + 命中状态 + 贡献方向 + 详情按钮 │
├─ 历史对比/补充信息 ─────────────────────┤
│ 近 N 次同申请人分数变化、数据来源时间、备注  │
└────────────────────────────────────────┘
```

颜色按"风险语义"而非原始数值着色：欺诈分高=红、信用分/综合分低=红，统一"红=危险"。

### 2.2 欺诈分（智察分）展开详情

**基础信息**
- 分数：22 分（当前示例）
- 等级：低
- 区间：0 ~ 100 分，阈值 ≥60 为高风险
- 含义：该申请人当前欺诈风险低
- 模型版本：AntiFraud-v3.2
- 计算时间：2026-07-19 10:23:14

**影响因素（按贡献度降序）**

| 因子 | 状态 | 对分数的影响 | 说明 |
|---|---|---|---|
| 身份信息一致性 | 正常 | 低风险 | 姓名/身份证/手机号/银行卡四要素一致 |
| 设备指纹风险 | 正常 | 低风险 | 当前设备未命中黑名单，无群控/模拟器特征 |
| 申请行为异常 | 需关注 | 中等风险 | 近 7 天申请频次偏高（5 次） |
| 多头借贷 | 需关注 | 中等风险 | 当前 3 家机构在查 |
| 黑名单/灰名单 | 未命中 | 低风险 | 未命中设备、身份证、手机黑名单 |
| 活体/人脸比对 | 通过 | 低风险 | 活体检测通过，人脸比对一致 |

> 复核提示：欺诈分低，但"多头借贷"和"申请行为异常"需结合信用分一起看；若申请人频繁申贷，需关注其真实资金需求。

### 2.3 信用分（智信分）展开详情

**基础信息**
- 分数：526 分（当前示例）
- 等级：弱
- 区间：300 ~ 900 分，阈值 ≥720 为优
- 含义：违约概率较高，信用资质偏弱
- 模型版本：CreditScore-v2.8
- 计算时间：2026-07-19 10:23:14

**维度拆解（雷达图或分项进度条）**

| 维度 | 权重 | 分项评分 | 状态 | 说明 |
|---|---|---|---|---|
| 还款历史 | 高 | 较低 | 偏弱 | 历史贷款存在逾期记录 / 或信用历史短 |
| 负债水平 | 高 | 低 | 弱 | 当前负债收入比偏高 |
| 查询次数 | 中 | 低 | 弱 | 近 6 个月硬查询 19 次，超过阈值 |
| 收入稳定性 | 中 | 中等 | 一般 | 收入流水存在波动 |
| 职业/居住稳定性 | 低 | 中等 | 一般 | 工作/居住时长一般 |

> 复核提示：信用分主要受负债水平和近期查询次数拖累。复核时可重点核实收入证明和近期申贷原因。

### 2.4 综合分（智融分）展开详情

**基础信息**
- 分数：704 分（当前示例）
- 等级：良
- 区间：300 ~ 900 分，阈值 ≥720 为优
- 含义：综合风险可控，但略低于优质线
- 模型版本：BlendScore-v1.5
- 计算时间：2026-07-19 10:23:14

**融合逻辑**

```
综合分 = f(欺诈分, 信用分, 产品策略调整)

- 欺诈分贡献：22 分（低）→ 正向
- 信用分贡献：526 分（弱）→ 负向
- 产品策略调整：消费分期产品，对欺诈容忍度低，对信用弱项适度加权
- 最终综合分：704 分（良）
```

**分项说明**

| 输入项 | 取值/等级 | 对综合分的影响 |
|---|---|---|
| 欺诈分 | 22 / 低 | 拉高综合分 |
| 信用分 | 526 / 弱 | 拉低综合分 |
| 产品风险系数 | 消费分期中低风险 | 中性偏正 |
| 渠道风险系数 | App 申请 | 中性 |
| 策略调整 | 无特殊调整 | 中性 |

> 复核提示：综合分处于"良"区间，主要因为信用分偏低但欺诈分很低。复核员可结合人工收入核实决定是否准入或降额。

## 2.5 预警阈值与决策阈值设计【优化】

### 2.5.1 阈值分两层

| 层 | 名称 | 作用 | 是否阻断决策 |
|---|---|---|---|
| 展示层 | **预警阈值（黄区）** | 颜色分档、提示关注 | 否，仅标记 |
| 决策层 | **决策阈值（红区）** | 触发自动拒绝 / 转人工 | 是，进入决策流 |

> 同一个分数通常有两条线：黄线（预警，提醒复核员关注）与红线（决策，触发动作）。两者含义不同，不能混。

### 2.5.2 各分数阈值档位（默认参考，可按产品覆盖）

**欺诈分（0~100，越高越危险）**

| 档 | 区间 | 颜色 | 含义与动作 |
|---|---|---|---|
| 低 | 0~30 | 绿 | 正常，进入后续流程 |
| 中（预警） | 30~60 | 黄 | 需关注，建议结合信用/负债；可准入但限额度，或转人工加核 |
| 高（决策） | 60~100 | 红 | 高风险，通常自动拒绝；若业务允许转人工强核 |

**信用分（300~900，越高越安全）**

| 档 | 区间 | 颜色 | 含义与动作 |
|---|---|---|---|
| 弱（预警） | 300~580 | 红 | 资质弱，建议拒绝或转人工；若准入须强担保/降额 |
| 中 | 580~720 | 黄 | 中等，可准入但降额/提利率/转人工 |
| 优 | 720~900 | 绿 | 优质，标准准入 |

**综合分（300~900，越高越安全）**：档位逻辑同信用分，作为最终参考。

### 2.5.3 阈值治理要求

- **配置位置**：阈值存于「产品配置 / 策略中心」，非全局硬编码；不同产品套不同阈值包（高风险产品阈值更严）。
- **变更管控**：修改阈值须留版本、走审批、写审计日志；支持冠军/挑战者（champion/challenger）灰度验证。
- **防漂移**：模型或数据源变更后须回测阈值有效性，防止阈值随分布漂移失效。
- **复核可见**：复核员在详情页进度条上能看到"本单命中哪条阈值线"（阈值刻度线），并可对阈值触发项标注"误命中 / 已复核"。
- **与决策流联动**：黄线仅标记，红线须映射到决策流的具体分支（拒绝节点 / 转人工节点），与「策略中心-决策流」可追溯联动。

## 2.6 综合分方向选型：扣分制(方案1) vs 累加制(方案2)【优化·待确认】

> 用户提问：综合分用"扣分制（越高越安全）"还是"累加制（越高越风险）"更合理、更权威？

### 2.6.1 两种方案本质

- **方案1（扣分制）**：起点总分，不达标项扣分，分数越低越安全 → 本质是 **goodness / 资质分**（分数高=好客户）。
- **方案2（累加制）**：每项风险得对应分，累加得风险值，分数越高风险越高 → 本质是 **risk index / 风险分**（分数高=风险高）。

### 2.6.2 行业权威惯例

- **信用/资质类评分普遍采用方案1（higher=better）**：FICO(300-850)、芝麻分(350-950)、央行征信评分、Basel II IRB 评分卡（PD 随分数升高而下降）。最权威的建模方法是**评分卡(scorecard)**：基础分 + Σ 属性分，good 属性加分、bad 属性减分，最终分数单调随"好"上升。同盾智信分(300-900)、智融分(300-900)也采用 higher=better。
- **欺诈/风险类评分普遍采用方案2（higher=worse）**：欺诈分(智察分 0-100)、设备风险分、内部风险指数。决策引擎中"命中规则按权重累加"天然就是方案2。

### 2.6.3 结论：非二选一，分别落位

两个方案对应两种语义视角，应分别用在正确的位置：

| 指标 | 采用方案 | 原因 |
|---|---|---|
| 综合分 / 信用分（对外资质结论） | **方案1（higher=safer）** | 更权威、业务方直觉一致、与智信分/智融分统一 |
| 风险指数（头部一眼结论） | **方案2（higher=riskier）** | 语义即"风险"，与规则权重累加天然一致 |
| 欺诈分 | **方案2（higher=riskier）** | 欺诈信号本就是累加的风险 |

### 2.6.4 推荐的引擎实现（两套贯通，不冲突）

```
引擎内部：所有命中规则/模型按权重 累加风险分(方案2) → RiskRaw(0~100, 越高越危险)
           ↓ 单调变换
对外展示：信用风格 综合分(方案1, 300~900, 越高越安全)
```

- 这样既保留方案2"规则累加"的工程自然性（命中即加权重，与「命中规则」卡片直接对应），又获得方案1"分数高=好客户"的业务可读性。
- 风险指数(方案2)与综合分(方案1)通过单调变换一一对应，复核员在两个视角间切换不会矛盾。

### 2.6.5 待用户拍板

- [ ] 风险指数是否确认采用方案2（higher=riskier，=规则权重累加）？

## 2.7 实现细节：具体公式与页面展示规范【优化·核心落地点】

> 将 §2.6 的"引擎内累加风险(方案2) → 变换为信用综合分(方案1)"落成可实现的公式与展示规范。

### 2.7.1 风险指数 RiskRaw（方案2：规则权重累加，越高越危险）

**输入项**
- 欺诈分 FraudScore ∈ [0,100]（higher=worse，本身即风险量纲）
- 命中规则集：每条规则 i 有 风险权重 wᵢ（0~100，业务配置）、命中状态 hᵢ、严重度 severityᵢ ∈ [0,1]
- 其他模型风险贡献 ModelRisk（可选）

**公式**
```
RulePoints = Σᵢ ( hᵢ × wᵢ × severityᵢ )                // 命中规则的加权累加
RiskRaw   = clamp( α·FraudScore + β·RulePoints + γ·ModelRisk , 0, 100 )
α + β + γ = 1      // 各分量权重，按产品配置（默认 α=0.5, β=0.5, γ=0）
```

**severity（严重度）取法**
- 二值规则（如设备黑名单命中）：hᵢ=1 时 severity=1.0，未命中贡献 0。
- 梯度规则（如多头借贷、查询次数）：severity = min(1, (实际值−阈值)/(封顶值−阈值))，或查表分级（例：2~3家=0.5、4~5家=0.8、6+=1.0）。

**与阈值联动（见 2.5）**：RiskRaw 0~30 绿(低) ｜ 30~60 黄(预警) ｜ 60~100 红(决策/高风险)

### 2.7.2 综合分 Composite（方案1：变换后资质分，越高越安全）

**变换链路**
```
Safety     = 1 − RiskRaw / 100                      // 风险安全度 0~1, higher=safer
CreditNorm = (信用分 − 300) / 600                   // 信用度 0~1
Blend      = β_c·CreditNorm + β_r·Safety            // β_c+β_r=1 (默认 0.6/0.4)
Blend'     = clamp( Blend × 产品系数 , 0, 1 )        // 产品策略微调
Composite  = round( 300 + 600 × Blend' )           // 映射到 300~900
```
- 单调性保证：RiskRaw↑ ⇒ Safety↓ ⇒ Composite↓；信用分↑ ⇒ Composite↑。两视角方向一致、不冲突。
- 产品系数默认 1.0（中性），高风险产品可 <1 收紧、低风险产品可 >1 放宽，存产品配置。

**与阈值联动（见 2.5）**：Composite 300~580 红(弱) ｜ 580~720 黄(中) ｜ 720~900 绿(优)

### 2.7.3 工作示例（与 §3.3 命中规则对齐）

输入（取文档示例申请人）：欺诈分=22；信用分=526；命中规则——
- 多头借贷 3家(阈2)：w=20，severity=(3−2)/(6−2)=0.25 → +5.0
- 查询19次(阈15)：w=20，severity=(19−15)/(30−15)=0.27 → +5.4
- DTI超阈：w=40，severity=1.0 → +40.0
- 设备黑名单 / 行业灰名单：未命中 → 0
- RulePoints = 50.4

计算：
- RiskRaw = 0.5×22 + 0.5×50.4 = 36.2 → 落入黄区(预警)
- Safety = 0.638；CreditNorm = (526−300)/600 = 0.377
- Blend = 0.6×0.377 + 0.4×0.638 = 0.481；产品系数 1.0
- Composite = 300 + 600×0.481 = 588.6 → **589（黄区·中）**

> 注：本节公式为权威落地口径。**§2.4 中"综合分=704(良)"为早期示意值，与本公式不自洽**（信用分仅 526 偏弱、规则中度命中，综合分应落入黄区），应以本节公式重新标定；后续报告与决策流均引用本节口径。

- 方向标签固定展示（↑越高越危险 / ↑越高越优质），消除三分数反向误读。
- 单栏纵向堆叠（沿用"从上到下"布局偏好）；综合分/欺诈分 作为风险指数的拆解，默认展开关键项、折叠未命中项。
- 所有展示数值与「决策流路径」可追溯联动（同一次申请的 RiskRaw、Composite 一致）。

### 2.7.5 权重与系数治理
- α,β,γ（风险指数分量权重）、β_c,β_r（综合分融合权重）、产品系数 均存「策略中心/产品配置」，按产品差异化，非硬编码。
- 修改须版本化 + 审批 + 审计日志；支持冠军/挑战者灰度；模型/数据源变更后回测防漂移。
- 权重变更影响历史案件可比性，须标注"生效时间"，报表按生效版本切分。

### 3.3 当前示例中各规则展开内容

#### 1）反欺诈-设备指纹黑名单
- **命中值**：未命中
- **阈值**：命中即触发
- **权重**：高
- **规则说明**：检查当前申请设备是否出现在已知欺诈设备黑名单中（如群控设备、模拟器、改机设备等）。
- **数据来源**：设备指纹服务 + 黑名单库
- **风险含义**：未命中表示当前设备未关联已知欺诈行为。
- **复核建议**：无异常，可忽略。

#### 2）反欺诈-多头借贷监测
- **命中值**：3 家机构在查
- **阈值**：≥2 家机构触发关注
- **权重**：中
- **规则说明**：监测近 30 天内查询申请人征信或授信的信贷机构数量。
- **数据来源**：外部多头数据服务
- **风险含义**：短期内被多家机构查询，可能存在资金饥渴或多头借贷风险。
- **复核建议**：关注收入负债比；若收入不足，建议降额或补充材料。

#### 3）信用-近 6 月查询次数
- **命中值**：19 次
- **阈值**：≥15 次触发关注
- **权重**：中
- **规则说明**：统计近 6 个月内对申请人的硬查询次数（贷款审批、信用卡审批类）。
- **数据来源**：征信数据
- **风险含义**：查询次数过多通常意味着申请人近期频繁寻求信贷，可能资金紧张。
- **复核建议**：要求申请人说明近期申贷原因；结合信用分判断。

#### 4）策略-收入负债比阈值
- **命中值**：超阈
- **阈值**：DTI ≥ 50% 触发
- **权重**：高
- **规则说明**：计算申请人月收入中用于偿还债务的比例（Debt-to-Income Ratio）。
- **数据来源**：申请人填写收入 + 外部负债数据
- **风险含义**：收入负债比过高，意味着可支配现金流不足，违约概率上升。
- **复核建议**：重点复核收入证明真实性；可考虑拒绝或大幅降低额度。

#### 5）名单-行业灰名单
- **命中值**：未命中
- **阈值**：命中即触发
- **权重**：低
- **规则说明**：检查申请人所属行业是否命中内部风险行业灰名单（如高波动行业、受限行业等）。
- **数据来源**：内部名单库
- **风险含义**：未命中表示行业无特殊风险。
- **复核建议**：无异常，可忽略。

## 4. 复核员视角的查看建议

### 4.1 按决策结果的关注重点

| 决策结果 | 复核重点 | 必看卡片 |
|---|---|---|
| 准入 | 确认无遗漏的高风险命中；核实额度/利率建议是否合理 | 决策结论、命中规则（高权重）、风险评分 |
| 拒绝 | 找到导致拒绝的核心规则/评分；确认拒绝理由合规 | 命中规则（高权重项）、信用分、综合分 |
| 转人工 | 判断机器无把握的原因；决定是否需要补充材料 | 风险评分各因子、命中规则、基础信息 |

### 4.2 常见复核场景

- **欺诈分低 + 信用分低**：可能是真用户但资质差，重点核收入/负债。
- **欺诈分高 + 信用分高**：存在伪冒风险，重点核身份/设备/人脸。
- **综合分与信用分差异大**：通常是欺诈分拉高/拉低，复核时要拆分看。
- **高权重规则命中**：优先看权重为"高"的规则，低权重未命中可忽略。
