## 一、信用风控列表页

### 1.1 页面概述

信用风控列表页是贷前审核的核心工作台，展示所有申请记录的信用评估结果。与信息核验列表不同，本列表聚焦于**信用风险评分**和**授信决策**，帮助风控人员快速定位高风险申请并做出授信判断。

### 1.2 页面样式参考

参考信息核验列表页（截图1），保持相同的顶部导航、左侧菜单、统计卡片、筛选区域和表格布局。

### 1.3 顶部统计卡片

| 统计项 | 说明 | 样例数据 |
|--------|------|----------|
| **待人工审核** | 信用评分处于中高风险区间，需人工介入审核的案件数 | **6** |
| **自动审批通过率** | 系统自动审批通过的申请占比 | **29%** |
| **待双人复核** | 已提交双人复核等待终审的案件数 | **1** |
| **高风险预警** | 信用评分≥60分的高风险案件数 | **1** |

### 2.4 报告分段说明

#### 第一段：信用评分总览

| 项目 | 说明 | 样例 |
|------|------|------|
| **信用评分** | 信用风险量化评分(0-100)，分数越高风险越高 | **88分** |
| **信用风险等级** | 根据评分划分：低风险(0-39)、中风险(40-59)、高风险(60-79)、极高风险(80-100) | **高风险** |
| **自动审批结果** | 系统基于评分的自动审批结果 | **拒绝** |
| **授信建议** | 系统给出的授信建议 | **建议拒绝，风险过高** |
| **评分维度分布** | 六大维度得分卡片 | 见下表 |

**六大维度得分卡片：**

| 维度 | 得分 | 权重 | 等级 | 说明 |
|------|:---:|:---:|:---|:---|
| 身份真实性 | 92 | 20% | 低 | 公安实名、银行卡四要素、活体检测均通过 |
| 还款能力 | 65 | 25% | 中 | 月收入25,000元，负债率约35% |
| 信用历史 | 38 | 25% | 低 | 跨行业联防联控库命中1条历史逾期 |
| 行为稳定性 | 55 | 10% | 中 | 手机号入网仅21天，不足30天 |
| 设备安全性 | 28 | 10% | 低 | 设备存在Root/越狱，关联多身份 |
| 关联风险 | 48 | 10% | 中 | 关联设备存在风险，疑似号码共用 |

**信用评分计算公式：**

```
信用评分 = 身份真实性×20% + 还款能力×25% + 信用历史×25%
         + 行为稳定性×10% + 设备安全性×10% + 关联风险×10%

         = 92×0.20 + 65×0.25 + 38×0.25 + 55×0.10 + 28×0.10 + 48×0.10
         = 18.4 + 16.25 + 9.5 + 5.5 + 2.8 + 4.8
         = 57.25
```

> 注：基础加权得分为57.25分（中风险），但实际显示88分（高风险）。说明存在**叠加惩罚机制**：当多个维度同时出现中高风险时，系统会额外加分。

**叠加惩罚机制：**

| 惩罚条件 | 加分 | 本例是否触发 |
|:---|:---:|:---:|
| 信用历史得分<40（存在逾期） | +10 | ✅ 触发 |
| 设备安全性得分<40（设备异常） | +10 | ✅ 触发 |
| 关联风险得分<50（关联风险） | +5 | ✅ 触发 |
| 行为稳定性得分<60（行为异常） | +5 | ✅ 触发 |
| 三个及以上维度得分<60 | +5 | ✅ 触发 |

```
最终评分 = 57.25 + 10 + 10 + 5 + 5 + 5 = 92.25 → 取整 88-92分
```

## 三、信用评分计算流程

```
申请人提交申请
    │
    ▼
[数据采集]
    ├── 身份信息：姓名、身份证、手机号、银行卡
    ├── 设备信息：设备指纹、Root状态、IP、GPS
    ├── 行为信息：操作轨迹、填写速度、页面停留
    ├── 征信信息：查询记录、逾期记录、多头借贷
    └── 资产信息：收入、房产、车辆、工作
    │
    ▼
[六大维度评分]
    ├── 身份真实性 = Σ(核验项得分 × 权重)
    ├── 还款能力 = Σ(评估项得分 × 权重)
    ├── 信用历史 = Σ(评估项得分 × 权重)
    ├── 行为稳定性 = Σ(评估项得分 × 权重)
    ├── 设备安全性 = Σ(评估项得分 × 权重)
    └── 关联风险 = Σ(评估项得分 × 权重)
    │
    ▼
[基础加权得分]
    基础分 = 身份真实性×20% + 还款能力×25% + 信用历史×25%
           + 行为稳定性×10% + 设备安全性×10% + 关联风险×10%
    │
    ▼
[叠加惩罚机制]
    ├── 信用历史得分<40：+10分
    ├── 设备安全性得分<40：+10分
    ├── 关联风险得分<50：+5分
    ├── 行为稳定性得分<60：+5分
    └── 三个及以上维度得分<60：+5分
    │
    ▼
[最终信用评分]
    最终评分 = min(100, 基础分 + 叠加惩罚分)
    │
    ▼
[自动审批决策]
    ├── 0-39分：通过
    ├── 40-59分：预警
    └── 60-100分：拒绝
    │
    ▼
[人工审核流程]
    ├── 通过/拒绝：待确认 → 人工确认
    ├── 预警：待审核 → 人工审核（通过/拒绝/限制额度/提交复核）
    └── 双人复核：已提交双人复核 → 终审决策
    │
    ▼
[授信决策]
    ├── 通过：按参考额度授信
    ├── 限制额度：按限制额度授信
    └── 拒绝：拒绝申请
```

## 四、信用风控 vs 欺诈识别 vs 信息核验 核心差异

| 对比维度 | 信息核验 | 欺诈识别 | 信用风控 |
|----------|----------|----------|----------|
| **核心目标** | 验证身份和信息真实性 | 识别欺诈行为和意图 | 评估信用风险和还款能力 |
| **评分方向** | 通过/拒绝/预警 | 越高越危险（欺诈） | 越高越危险（信用） |
| **满分含义** | — | 100分=极高欺诈风险 | 100分=极高信用风险 |
| **数据来源** | 公安库、银行卡、运营商 | 设备指纹、黑名单、团伙库 | 征信报告、收入、资产、历史还款 |
| **自动决策** | 通过/拒绝/预警 | 通过/拒绝/转人工/观察 | 通过/预警/拒绝 |
| **人工处置** | 确认/复核/放行 | 确认欺诈/误判放行/加入黑名单 | 审核通过/拒绝/限制额度 |
| **输出产物** | 核验结果、异常值 | 欺诈评分、规则命中、团伙关联 | 信用评分、维度得分、授信建议 |
| **关注重点** | "这个人是谁？" | "这个人是不是骗子？" | "这个人能不能还钱？" |
