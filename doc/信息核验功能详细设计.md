# 信息核验 · 功能详细设计

> 整理日期：2026-07-19
> 位置：零售信贷风控 > 贷前审核 > 信息核验
> 范围：模块定位、人工/自动定性、核验能力、业务流程、详情页、角色权限、与申贷审核联动
> 标注：【需求】= 来自既有功能定义；【补充】= 参考行业补齐的合理设计。

---

## 1. 模块定位与边界

### 1.1 一句话定位

信息核验是贷前审核中**"信息层 + 身份层"的验真环节**——只回答"申请人填的信息格式对不对、是不是本人、材料真不真"，不做信用好坏、欺诈团伙、评分高低的判断。

### 1.2 与相邻模块的边界（避免混淆）

| 模块 | 管什么 | 不管什么 |
|---|---|---|
| **信息核验**（本模块） | 格式校验、身份验真（三/四要素、人脸活体）、材料真伪、字段逻辑一致性 | 信用、欺诈团伙、评分 |
| 信用风控 | 多头借贷、共债、联防联控 | 身份是否本人 |
| 欺诈识别 | 设备指纹、团伙、行为异常 | 填写格式对错 |
| 申贷审核 | 调度以上全部能力，出整体决策 | 单项能力的内部实现 |

> 关键：信息核验是**申贷审核决策流里的一个前置环节/能力**，其结论作为结构化字段回流给决策引擎。它既作为「申贷审核详情页」里的一个区块被展示，也独立成一个菜单页用于**配置、观测、异常件处置**。

## 2. 核心定性：核验是人工还是自动？

### 2.1 结论

**自动为主、人工兜底（Straight-Through Processing + 异常件人工复核）。**

- **绝大多数核验项天然是自动的**：格式校验、三/四要素核验、活体检测、人脸比对、OCR 识别，全部由系统调用接口/模型实时完成，无需人工。
- **纯人工不该做**：贷前进件量大（日均数千到数十万），逐笔人工验真不现实，且人工验真身份的准确率反而不如活体+人脸比对。
- **但不能纯自动**：存在自动无法拍板的"灰色地带"，必须有人工兜底，否则要么误拒真人、要么放过伪冒。

### 2.2 什么时候必须人工介入（异常件触发条件）

| 触发场景 | 原因 | 人工要做什么 |
|---|---|---|
| OCR 置信度低 | 证件照模糊/反光/遮挡，识别不确定 | 人工核对证件原图与识别结果 |
| 人脸比对相似度处于临界区间 | 既不能判同一人，也不能判非本人 | 人工目视比对活体照与证件照 |
| 要素核验返回"无法核实" | 数据源无记录/超时，非"不一致" | 人工决定是否放行或要求补充 |
| 字段逻辑矛盾 | 如年龄与身份证不符、地址前后不一 | 人工判断是笔误还是造假 |
| 疑似证件伪造/篡改 | 系统检出 PS 痕迹/翻拍 | 人工研判，必要时上报欺诈 |
| 材料缺失 | 必要证件未上传或不清晰 | 发起补充材料请求（对客） |

> 设计要点：自动核验对每一项输出 **通过 / 不通过 / 待人工确认** 三态，而不是简单的通过/不通过二态。凡落入"待人工确认"的，进入**核验异常件队列**由人处理。

### 2.3 核验模式配置（可按产品/渠道调节严格度）

- **宽松模式**：仅格式+要素核验，人脸活体可选，适合小额、低风险产品。
- **标准模式**：格式+四要素+活体+人脸比对，临界值转人工。
- **严格模式**：全项核验 + 证件防伪检测 + 强制人工抽检比例，适合大额/抵押类。
- 严格度由风控管理员在配置页按产品/渠道设定，直接影响"转人工"比例。

## 3. 核验能力清单

| 能力 | 类型 | 自动/人工 | 输出 |
|---|---|---|---|
| 姓名/身份证/手机号/银行卡格式校验 | 格式层 | 纯自动 | 通过/不通过 |
| 身份二要素（姓名+身份证） | 身份层 | 纯自动 | 一致/不一致/无法核实 |
| 银行卡三要素（+银行卡号） | 身份层 | 纯自动 | 一致/不一致/无法核实 |
| 手机四要素（+手机号） | 身份层 | 纯自动 | 一致/不一致/无法核实 |
| 证件 OCR 识别 | 材料层 | 自动（低置信转人工） | 结构化字段 + 置信度 |
| 活体检测 | 身份层 | 纯自动 | 通过/不通过 |
| 人脸比对（活体 vs 证件） | 身份层 | 自动（临界转人工） | 相似度分 + 判定 |
| 证件防伪/篡改检测 | 材料层 | 自动（疑似转人工） | 正常/疑似伪造 |
| 字段逻辑一致性校验 | 逻辑层 | 自动（矛盾转人工） | 一致/矛盾清单 |

## 4. 业务流程设计

### 4.1 自动核验主流程（Straight-Through）

```
申请提交
   │
   ▼
① 格式校验 ──不通过──▶ 打回/提示重填（对客）
   │通过
   ▼
② OCR 识别证件 ──低置信──▶ [异常件队列]
   │正常
   ▼
③ 要素核验(二/三/四要素) ──不一致──▶ 核验不通过（回流引擎）
   │一致 / 无法核实 ──▶ [异常件队列]
   ▼
④ 活体检测 + 人脸比对 ──不通过──▶ 核验不通过（回流引擎）
   │临界相似度 ──▶ [异常件队列]
   │通过
   ▼
⑤ 证件防伪 + 逻辑一致性 ──疑似/矛盾──▶ [异常件队列]
   │正常
   ▼
核验通过 → 结构化结论回流申贷审核决策引擎
```

### 4.2 异常件人工处置流程

```
[异常件队列]
   │ 复核员认领 / 系统分配
   ▼
打开核验处置台
   │
   ├─ 核对证件原图 vs OCR 结果
   ├─ 目视比对活体照 vs 证件照
   ├─ 判断字段矛盾是笔误还是造假
   │
   ▼
处置决策
   ├─ 核验通过 ──▶ 结论回流引擎（标记：人工确认通过）
   ├─ 核验不通过 ──▶ 结论回流引擎（标记：人工判定不通过）
   ├─ 要求补充材料 ──▶ 对客发起补件（申请人上传后重新核验）
   └─ 疑似欺诈上报 ──▶ 推送欺诈识别模块 + 加入名单
   │
   ▼
全程写入审计日志（操作人/时间/结论/理由）
```

### 4.3 与申贷审核的时序关系

- 信息核验是申贷审核决策流的**前置节点**：核验未过 → 决策流通常直接判"拒绝/转人工"，不再往下跑信用/欺诈。
- 核验"待人工确认"时，该笔申贷审核状态可表现为**审核中**（等核验结论）或直接**转人工**（并入申贷审核的待人工复核，共用复核记录时间线）。
- 核验结论以结构化字段（各项结果 + 置信度 + 人工标记）回流，成为决策报告"信息核验结论"模块的数据源。

## 5. 详情页设计（核验结果详情）

从「信息核验」列表或「申贷审核详情页」的信息核验区块点入。采用**单栏纵向卡片**（沿用你确认的从上到下布局）。

### 5.1 页面结构

```
┌─ 顶部：申请编号 · 申请人(脱敏) · 核验总结论徽章(通过/不通过/待人工) ─┐
├─ 卡片1 核验结论总览 ────────────────────────────┤
│  总结论 · 核验模式(宽松/标准/严格) · 核验时间 · 处理方式(自动/人工) │
├─ 卡片2 格式校验 ───────────────────────────────┤
│  姓名/身份证/手机号/银行卡 逐项：值(脱敏) + 结果 + 异常提示        │
├─ 卡片3 身份要素核验 ───────────────────────────┤
│  二/三/四要素：核验项 + 结果(一致/不一致/无法核实) + 数据时间     │
├─ 卡片4 活体与人脸比对 ─────────────────────────┤
│  活体结果 + 人脸相似度分 + 阈值 + 判定；活体照/证件照并排(脱敏)   │
├─ 卡片5 证件 OCR 与防伪 ────────────────────────┤
│  证件影像 + OCR字段 + 置信度 + 防伪检测结果                     │
├─ 卡片6 字段逻辑一致性 ─────────────────────────┤
│  矛盾清单(字段A vs 字段B) + 说明；无矛盾则显示"通过"           │
├─ 卡片7 人工处置记录(若经人工) ─────────────────┤
│  复核员 + 处置动作 + 理由 + 时间(多轮时间线,复用复核记录组件)    │
└─ 底部操作区：通过 / 不通过 / 要求补充材料 / 疑似欺诈上报 / 导出 ──┘
```

### 5.2 关键字段说明

| 卡片 | 关键字段 | 复核价值 |
|---|---|---|
| 核验结论总览 | 处理方式=自动/人工、核验模式 | 一眼知道这单是机器过的还是人过的 |
| 身份要素核验 | "无法核实" vs "不一致" 要分开 | 无法核实≠造假，处置方式不同 |
| 活体人脸 | 相似度分 + 阈值 | 临界值时人工据此目视复核 |
| OCR 与防伪 | 置信度 + 防伪结果 | 低置信/疑似伪造是人工介入主因 |
| 逻辑一致性 | 矛盾清单 | 定位笔误还是造假 |

### 5.3 操作按钮（按结论/权限动态显示）

| 场景 | 显示按钮 |
|---|---|
| 待人工确认 | 通过 / 不通过 / 要求补充材料 / 疑似欺诈上报 |
| 已自动通过 | 查看详情 / 导出 |
| 已人工处置 | 查看处置记录 / 导出 |

> 所有人工处置写入「企业设置-审计日志」，理由必填。

## 6. 角色与权限

| 角色 | 在信息核验模块的职责 | 可见/可操作 |
|---|---|---|
| **风控管理员** | 配置核验规则、要素类型、活体/人脸阈值、核验模式、抽检比例 | 配置页 + 全量记录 + 大盘 |
| **信审员/复核员** | 处置异常件队列，逐笔人工核对与判定 | 异常件队列 + 处置台 + 本人处理记录 |
| **风控主管** | 抽检、疑难件升级审批、翻转率监控 | 队列 + 抽检 + 大盘 |
| **审计员** | 查核验记录与审计日志（只读） | 记录 + 审计日志（只读） |

## 7. 页面清单（作为独立菜单时）

| 页面 | 主要功能 |
|---|---|
| 核验概览（大盘） | 核验通过率、转人工率、各失败原因分布、按产品/渠道对比 |
| 异常件队列 | 待人工确认列表 + 认领/分配 + 优先级 + SLA 计时 |
| 核验处置台 | 单笔异常件处置（第 5 节详情页 + 操作） |
| 核验记录查询 | 全量核验记录检索（按申请人/时间/结果/模式），钻取详情 |
| 核验规则配置 | 要素类型、阈值、核验模式、抽检比例、按产品/渠道差异化 |
