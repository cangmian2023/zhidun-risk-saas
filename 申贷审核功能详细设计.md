# 申贷审核 · 功能详细设计

> 整理日期：2026-07-19
> 范围：零售信贷风控 > 申贷审核（原"进件审核（风险评估）"）模块的列表、详情、自动化报告、决策结果后续操作的详细设计
> 说明：以业务需求为种子，参考行业信贷审核系统（同盾天策决策引擎、Fero 风控引擎、银行信审系统）补全必要的交互与状态设计。
> 标注：【需求】= 来自本次明确诉求；【补充】= 参考行业补齐的合理设计。

---

## 1. 模块定位

申贷审核是零售信贷风控中**处理一笔贷款申请的总入口**：接收来自各渠道的申贷信息，调用决策引擎跑决策流（信息核验、信用风控、欺诈识别、评分），输出决策建议，并承载人工复核与报告生成。

它向下依赖「策略中心」（指标/规则/决策流/模型）与「评分产品」（智察分/智信分/智融分），向上输出决策结果与报告，是对外展示"这笔申请到底过没过、为什么"的核心页面。

本次细化聚焦四块：【需求】列表筛选 + 审核状态列、详情查看、按产品差异化报告、决策结果下一步操作。

## 2. 列表页设计

### 2.1 列表字段（列）【需求·核心列 + 补充列】

| 列 | 说明 | 来源/类型 | 排序 |
|---|---|---|---|
| 申请编号 | 唯一标识，可点击进详情 | 系统生成 | 是 |
| 申请人 | 姓名（脱敏，如 张\*三）+ 头像 | 申请信息 | 否 |
| 产品 | 信贷产品名称 | 企业设置-产品管理 | 是 |
| 渠道 | 申请来源（App/H5/小程序/线下/合作方） | 企业设置-渠道管理 | 是 |
| 申请额度 | 申贷金额（元） | 申请信息 | 是 |
| 欺诈分 | 智察分（0~100，越高风险越高） | 评分产品-智察分 | 是 |
| 信用分 | 智信分（300~900，越高违约概率越低） | 评分产品-智信分 | 是 |
| 决策结果 | 准入 / 拒绝 / 转人工 | 决策引擎输出 | 是 |
| 审核状态【需求·新增列】 | 待审核/审核中/已通过/已拒绝/转人工/已关闭 | 状态机 | 是 |
| 申请时间 | 提交时间 | 系统 | 是 |
| 操作 | 查看 / 复核（按状态显示） | - | 否 |

> 列支持自定义显示/隐藏、拖拽排序（【补充】），默认展示上表核心列。

### 2.2 筛选功能【需求】

顶部筛选区，按以下条件组合筛选（支持"且"关系，多条件叠加）：

| 筛选条件 | 控件类型 | 说明 |
|---|---|---|
| 产品 | 下拉多选（树形：产品大类-产品） | 来自企业设置的产品目录 |
| 渠道 | 下拉多选 | 来自企业设置的渠道目录 |
| 申请额度 | 数值区间（最小值~最大值） | 支持快捷区间：0-1万 / 1-5万 / 5-20万 / 20万以上 |
| 欺诈分 | 数值区间（0~100） | 支持快捷：低风险<30 / 中 30-60 / 高>60 |
| 信用分 | 数值区间（300~900） | 支持快捷：低<580 / 中 580-720 / 高>720 |
| 决策结果 | 单选/多选 | 准入 / 拒绝 / 转人工 |
| 审核状态【需求】 | 单选/多选 | 待审核/审核中/已通过/已拒绝/转人工/已关闭 |
| 申请时间 | 日期范围 | 今天/近7天/近30天/自定义 |
| 关键字 | 文本框 | 申请编号、申请人姓名、手机号（脱敏检索） |

- 筛选条件可保存为"常用筛选方案"（【补充】），如"高欺诈待复核""消费分期拒绝单"。
- 支持一键重置、筛选结果导出（CSV/Excel，受数据权限控制）。

### 2.3 审核状态定义与状态机【需求·新增】

审核状态贯穿一笔申请在申贷审核模块中的全生命周期，与"决策结果"是**两个正交维度**：

- **决策结果** = 系统/人工给出的结论标签（准入 / 拒绝 / 转人工）
- **审核状态** = 这笔申请当前处于流程的哪个节点

**状态枚举（可筛选值）：**

| 审核状态 | 含义 | 进入条件 | 可流转到 |
|---|---|---|---|
| 待审核 | 已提交，尚未进入自动审核 | 进件提交成功 | 审核中 |
| 审核中 | 决策引擎运行中，或人工复核处理中 | 触发自动审核 / 转人工后分配复核员 | 已通过 / 已拒绝 / 转人工 |
| 已通过 | 最终准入（自动或人工） | 自动准入 或 人工通过 | 已关闭（如放款失败/撤销） |
| 已拒绝 | 最终拒绝（自动或人工） | 自动拒绝 或 人工拒绝 | 已关闭 |
| 转人工 | 自动决策为转人工，等待人工给出最终结论 | 决策引擎输出"转人工" | 审核中（人工复核中）→ 已通过/已拒绝 |
| 已关闭 | 撤销/过期/客户放弃 | 任意终态后可关闭 | 终态 |

**状态机简图：**

```
提交 ──▶ 待审核 ──▶ 审核中 ──┬──▶ 已通过（决策结果=准入）
                              ├──▶ 已拒绝（决策结果=拒绝）
                              └──▶ 转人工 ──▶ 审核中(人工) ──┬──▶ 已通过
                                                                  └──▶ 已拒绝
任意终态 ──▶ 已关闭
```

> 设计要点：当审核状态=「转人工」时，决策结果固定为"转人工"；当审核状态=「已通过/已拒绝」时，决策结果对应"准入/拒绝"。这样列表里既能按"状态"筛（找待办），也能按"结论"筛（找某类结果），互不冲突。

### 2.4 列表操作与批量【补充】

- 行内操作：查看详情（始终）、复核（仅"待审核/转人工/审核中"显示）、导出报告（仅终态）。
- 批量操作：勾选多行 → 批量导出报告、批量转人工（高风险待办集中处理）、批量关闭。
- 分页：每页 10/20/50 条，支持跳页；大数据量下虚拟滚动。
- 实时刷新：审核中的单子结果返回后自动更新状态（WebSocket 推送，与「消息通知」联动）。

## 3. 详情页设计

点击列表"查看"或"申请编号"进入详情页。详情页采用**双栏/分段结构**：上方申请信息，下方审核信息与操作区。

### 3.1 申请详情【需求·申请信息】

展示申请人提交的全量信息，分卡片：

- **基础信息**：申请人（脱敏姓名/性别/年龄）、证件类型与号（脱敏）、手机号（脱敏）、产品、渠道、申请额度、期限、申请用途、申请时间。
- **证件与材料**：身份证正反面、银行卡、活体/人脸比对结果（与「信息核验」联动）。
- **填写信息**：表单各字段原值，格式异常处高亮标注（如身份证格式错误）。
- **设备与环境**：设备指纹、IP、定位、申请时段（与「欺诈识别」联动，标记群控/模拟器/异地）。
- **关联信息**：同设备历史申请数、同人历史进件（链接跳转）、多头借贷机构数（与「信用风控」联动）。

### 3.2 审核详情【需求·审核信息】

- **决策结论卡片**：决策结果（大字标注）、审核状态、最终额度/利率（如适用）、决策时间、处理人（系统/复核员）。
- **风险评分**：欺诈分（智察分）、信用分（智信分）、综合分（智融分）及各自等级与阈值线。
- **命中规则/模型**：本次决策流命中的规则清单（规则名、命中值、权重）、调用的模型/评分卡、名单命中（黑/灰/白）。
- **决策流路径**：可视化展示本次申请走过的决策流节点与分支（与「策略中心-决策流」联动，可追溯）。
- **人工复核记录**：若经人工，展示复核员、复核时间、复核结论、复核备注、复核 SLA 耗时。
- **决策报告**：一键查看/下载本次自动生成的报告（见第 4 节，按产品差异化）。

### 3.3 详情页操作按钮【需求·进一步操作】

按钮按当前审核状态与权限动态显示：

| 状态 | 显示按钮 | 动作 |
|---|---|---|
| 待审核 / 转人工 / 审核中(人工) | 通过 / 拒绝 / 挂起 | 人工给出最终结论（需填写理由，留痕） |
| 待审核 / 转人工 | 转人工复核 | 将自动单转人工，分配复核员 |
| 审核中(自动) | 查看进度 | 展示决策引擎运行进度 |
| 已通过 / 已拒绝 | 查看报告 / 导出报告 | 报告查看与下载 |
| 已通过 / 已拒绝 | 发起复议【补充】 | 客户申诉或内部复议入口（合规留痕） |
| 任意 | 查看关联进件 / 查看同设备历史 | 跳转关联查询 |
| 任意终态 | 关闭 | 关闭该申请（撤销/归档） |

> 所有人工操作写入「企业设置-审计日志」，满足合规追溯。

## 4. 自动化审核报告（按产品差异化）【需求】

### 4.1 报告模板机制

报告由决策引擎在得出决策结果后**自动生成**，不依赖人工撰写。核心机制：

- **报告模板与产品绑定**：每个信贷产品配置一套报告模板（产品管理中设置），定义"展示哪些模块、模块顺序、阈值标注方式、措辞口径"。
- **引擎按产品取模板**：决策完成后，引擎读取该申请所属产品的模板，组装对应内容生成报告（PDF/在线 HTML）。
- **模板可配置**：风控管理员可在「策略中心」或产品配置页编辑模板，无需改代码。

### 4.2 各产品报告差异（典型示例）

| 产品 | 报告侧重模块 | 差异化要点 |
|---|---|---|
| 现金贷（小额短期） | 欺诈分为主 | 精简版：欺诈分+设备指纹+多头借贷+命中黑名单；忽略长期信用，强调实时风险 |
| 消费分期 | 信用分+场景 | 信用分+消费场景核验+商户真实性+还款能力；展示授信额度建议 |
| 小微企业贷 | 企业风控+经营 | 法人关联+企业征信+经营流水+担保；报告含企业维度与法人维度双栏 |
| 房抵/抵押贷 | 资产+估值 | 房产证 OCR+估值+抵押物状态+负债率；报告含资产证明附件清单 |
| 汽车金融 | 车辆+反欺诈 | 车辆信息+GPS/抵押状态+身份反欺诈+首付来源 |

> 差异本质：不同产品的风险敞口不同 → 报告收集的"证据模块"和"权重呈现"不同。模板机制让同一套引擎产出不同口径的报告。

### 4.3 报告内容结构（通用骨架，模块按产品启停）

1. 报告头：申请编号、产品、渠道、生成时间、报告版本
2. 申请人画像：基础信息 + 风险分层标签
3. 风险评分区：欺诈分/信用分/综合分 + 等级 + 阈值线标注
4. 证据模块（按产品启用）：信息核验结论、信用风控（多头/共债）、欺诈识别（设备/团伙）、材料 OCR
5. 命中规则与模型：规则清单、模型/评分卡、名单
6. 决策建议：结论（准入/拒绝/转人工）+ 额度/利率建议 + 关键依据摘要
7. 附件与溯源：原始材料、决策流路径链接

## 5. 不同决策结果的下一步操作【需求】

详情页底部设**"下一步操作"面板**，根据决策结果动态展示不同内容与动作，引导审核人员/客户走完后续流程。

### 5.1 决策结果枚举

准入（通过） / 拒绝 / 转人工（三者与第 2.3 节状态机一致）。

### 5.2 下一步操作矩阵

| 决策结果 | 面向角色 | 下一步操作内容 | 系统动作 |
|---|---|---|---|
| **准入** | 客户 | 查看授信通知、确认额度/利率、签署电子合同 | 推送授信结果、生成合同待签、触发放款前置 |
| **准入** | 风控/运营 | 监控首笔用信、配置额度生效 | 写入额度系统、加入贷中监控名单 |
| **拒绝** | 客户 | 查看拒绝原因（合规须告知）、了解复议/申诉渠道、知悉冷静期（N 天内不可重复申请） | 推送拒绝通知（含原因摘要，不泄露模型细节）、记录拒绝原因码 |
| **拒绝** | 风控 | 标记拒绝原因归类、可选加入灰名单观察 | 拒绝原因统计、按需更新名单 |
| **转人工** | 复核员 | 查看复核工作台、调取申请+审核详情、给出通过/拒绝/挂起结论并填理由 | 分配复核员、启动复核 SLA 计时、结果回流决策引擎 |
| **转人工** | 客户 | 等待复核结果通知 | 推送"审核中"状态通知 |

### 5.3 操作面板交互【补充】

- 面板顶部显示当前决策结果大字标签与颜色（准入=绿/拒绝=红/转人工=黄，遵循国内金融红涨绿跌惯例需注意：此处"准入通过"建议用蓝/绿，"拒绝"用红，"转人工"用黄，避免与行情涨跌混淆，采用"通过=蓝"更稳妥）。
- 转人工时，面板内嵌"复核结论"表单：结论单选 + 理由必填 + 提交；提交后状态流转并更新列表。
- 拒绝时，面板展示"拒绝原因码 + 可告知客户的话术预览"，确保对外口径合规。
- 所有操作产生待办并推送到「消息通知」，复核员在消息中心可一键跳转复核。
